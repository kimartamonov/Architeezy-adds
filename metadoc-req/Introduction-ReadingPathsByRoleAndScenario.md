# Требования к странице: Маршруты чтения по ролям и сценариям

## Раздел
Введение

## Страница
Маршруты чтения по ролям и сценариям

## Назначение страницы
Помочь пользователю быстро выбрать нужный путь по документации в зависимости от роли, типа задачи и результата, который он хочет получить.

## Для кого эта страница
Все пользователи, пришедшие с конкретным кейсом.

## Что должно быть написано на странице
- карту ролей: enterprise, solution, business, data, infrastructure
- короткие маршруты чтения для типовых задач
- ссылки на базовые страницы и прикладные паттерны
- отдельный путь для быстрого пилота и для API/MCP сценариев

## На какие вопросы пользователя страница должна отвечать
- С чего начать именно мне?
- Какие страницы обязательны для моего кейса?
- Где искать примеры под мою роль?

## Каким пользователям эта страница особенно полезна
- архитекторы
- аналитики
- команды внедрения

## Какие сценарии эта страница должна покрывать
- enterprise-capability-portfolio
- integration-contracts
- process-role-document-sla
- data-product-governance
- infrastructure-topology-dr

## Ожидаемый результат после чтения
Пользователь должен понять, как применить содержимое страницы к своему кейсу и что именно он должен сделать следующим шагом в проектировании, проверке или использовании метамодели.
