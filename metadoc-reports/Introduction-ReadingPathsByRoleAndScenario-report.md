## Для каких ролей и сценариев написан документ

Документ написан для всех пользователей, которые приходят в документацию с конкретной задачей и хотят быстро выбрать правильный путь чтения.

Особенно ориентирован на:

- корпоративных архитекторов;
- системных и интеграционных архитекторов;
- бизнес-аналитиков и владельцев процессов;
- архитекторов данных и data governance команды;
- инфраструктурных и платформенных архитекторов.

## Что вошло в страницу

- общая логика чтения документации;
- быстрый выбор маршрута;
- отдельные маршруты по пяти основным ролям;
- маршруты по типу сценария;
- отдельный трек для быстрого пилота;
- отдельный трек для API, MCP и AI-сценариев;
- рекомендации для смешанных кейсов и командного использования документации.

## Сделанные допущения

- Страница сделана как навигационная карта, а не как детальное содержание всех следующих страниц.
- Внутренние ссылки пока указаны как названия страниц, чтобы потом их можно было легко привязать к готовым документам.
- Скриншоты оставлены как плейсхолдеры, чтобы их можно было добавить позже после согласования визуального формата.

## Открытые вопросы и риски

- Когда будет готово больше страниц документации, возможно, стоит добавить сокращённую таблицу с кликабельными переходами.
- Позже может понадобиться отдельный маршрут для внутренних авторов метамоделей и для потребителей готовых моделей.
