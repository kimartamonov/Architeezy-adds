# Что нужно в документации для кейса Интеграционный контракт

## Нужные разделы документации

### 1. Как создать DSL для интеграций

Хочу увидеть пошаговый пример метамодели, где есть системы, контракты, каналы, сообщения, схемы и SLA.

### 2. Как задавать разные типы взаимодействия

Нужны примеры для синхронных и асинхронных контрактов, чтобы не изобретать семантику заново.

### 3. Как описывать связи и технические атрибуты

Хочу увидеть, как правильно моделировать формат, auth, retry, version и ownership.

### 4. Как совместить диаграмму и реестр контрактов

Нужны инструкции по созданию визуальной схемы и табличного каталога на одной модели.

### 5. Как использовать API и MCP для такого языка

Хочу увидеть, как специализированную модель можно читать, создавать или обновлять через API/AI-инструменты.
